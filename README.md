# SlackMemberCheck 判定ロジックREADME

このプロジェクトは、フォーム回答者とSlackメンバーを突合し、
スプレッドシートのA列（加入準備WS参加チェックボックス）を更新します。

## 1. どのデータを使っているか

### 回答シート
- 対象シート名: フォームの回答 1
- 参照列:
  - A列: 加入準備WS参加（チェックボックス）
  - B列: タイムスタンプ
  - C列: メールアドレス
  - D列: お名前（漢字）

### Slack側
- API: users.list（ページネーション対応）
- 使用項目:
  - email
  - real_name（なければ profile.real_name、さらに無ければ user.name）

## 2. Slackメンバーとして扱う条件

Slack APIで取得したユーザーのうち、次は除外されます。

- deleted のユーザー
- is_bot のユーザー
- email が空のユーザー

この時点で除外された人は、最初から突合対象に入りません。

## 3. 正規化ルール

### メールアドレス
- trim（前後空白除去）
- 小文字化

### 氏名
- 半角/全角スペースを削除
- Unicode正規化（NFKC）
- trim

## 4. 判定ロジック（1行ごと）

各回答行に対して次の順で判定します。

1. A列が空欄でなければ、その値を保持して再判定しない
2. A列が空欄なら、C列メールをSlackメール集合と照合
3. メール不一致の場合のみ、D列氏名（正規化後）をSlack氏名集合と照合
4. 一致したら A列 = true、未一致なら A列 = false

判定優先順位は「メール一致 > 氏名一致」です。

## 5. 「対応するSlackメンバーが少なく見える」主な理由

件数が想定より少なく見える場合、主に次が原因です。

- Slack側で bot/deleted/emailなしのため除外されている
- フォーム側のメールとSlackメールが異なる（別メールで登録している）
- Slack表示名とフォーム氏名が、正規化後でも一致しない
  - 例: ニックネーム、旧姓、別表記
- A列が既に true/false で埋まっている行は再判定されない

## 6. 補助出力（Slack未対応シート）

別シート「Slack未対応」に、
「Slackにいるが回答シートで対応者が見つからないメンバー」を出力します。

- 照合順序は同じ（メール一致を優先、次に氏名一致）
- 出力列:
  - 最終更新
  - Slack表示名
  - Slackメールアドレス
  - 正規化氏名

0件の場合は「未対応者は0件です」と表示します。

## 7. まとめ（運用上のポイント）

- 判定の中心はメール一致
- 氏名一致は補助的なフォールバック
- A列が埋まった行は再判定されないため、再評価したい行はA列を空欄に戻してから実行する
