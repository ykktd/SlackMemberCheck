# Slack参加確認自動化ツール 設計書 (Google Apps Script)

## 1. 目的

入会フォームに回答したユーザーが、案内済みのSlackワークスペースに実際に参加しているかを自動判定し、スプレッドシート上のステータスを更新する。

## 2. 実行環境

- Google Apps Script (コンテナバインド型)
- 連携先: Slack Web API (users.list エンドポイント)
- 認証: Slack ボットトークン (xoxb-...) ※事前に chat:write, users:read, users:read.email スコープを持つボットを作成しておく

## 3. データ構造

### 入力データ（スプレッドシート: "フォーム回答" シート）

- A列: タイムスタンプ
- B列: 氏名（漢字フルネーム、姓名の間にスペースが含まれる可能性あり）
- C列: メールアドレス
- D列: Slack参加ステータス（プログラムが書き込む列）

### 取得データ（Slack API）

- real_name: Slack上の表示名
- email: Slackアカウントの登録メールアドレス

## 4. アルゴリズム詳細

プログラムは以下のステップで実行する。

### Step 1: Slackメンバー情報の取得

- Slack APIの users.list を叩き、全メンバーの real_name と email のペアをオブジェクト配列として取得する。
- ※ページネーション（cursor）に対応し、全ユーザーを取得すること。

### Step 2: データの正規化関数 (Normalize)

突合精度を上げるため、以下の処理を行う関数を実装する

- 引数: 文字列（氏名）
- 処理:
  1. 全ての全角・半角スペースを削除。
  2. Unicode正規化（NFKC）を行い、全角英数字を半角に、半角カタカナを全角に変換。
- 戻り値: 正規化された文字列

### Step 3: マッチングロジック

スプレッドシートの各行に対して以下の判定を行う。

1. メールアドレス照合: フォームのメールアドレスが、Slackのメンバーリストのいずれかの email と完全一致するか。
2. 氏名照合 (メール不一致の場合): フォームの氏名とSlackの real_name をそれぞれ「Step 2の関数」で正規化し、完全一致するか。

### Step 4: 結果の書き込み

- 一致した場合: D列に「参加済み」と記入。
- 一致しない場合: D列に「未参加」と記入。
- ※既存の「参加済み」を上書きしないよう、未設定の行のみ処理する、あるいは一括更新する。

## 5. 実装上の注意点

- APIトークンの管理: PropertiesService を使用して、スクリプトプロパティからトークンを取得するコードにすること。
- 実行速度: スプレッドシートへの書き込みは setValue を繰り返すのではなく、配列に格納して一括で setValues すること。
- エラーハンドリング: APIリクエスト失敗時の例外処理を含めること。
